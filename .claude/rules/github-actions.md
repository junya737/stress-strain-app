# GitHub Actions / Issue対応時の指針

## 回答形式
全ての回答は日本語で行い、以下の構成で提示すること：

1. **概要**: 問題点・原因・解決策を各1-2行で
2. **詳細分析**: 現状と期待動作の差異、根本原因、解決策の比較（複数案ある場合は表形式）
3. **実装内容**: 修正ファイル・箇所（`file_path:line_number`）・テスト追加
4. **コミットメッセージ案**: `type(scope): 概要` 形式、`Fixes: #issue番号`
5. **テスト方法**: 単体テスト・統合テスト・手動テストの手順
6. **影響範囲**: 直接・間接的影響、パフォーマンス、互換性
7. **注意点とリスク**: エッジケース、副作用、技術的負債

## PR作成時のルール
- ブランチ名: `fix-issue-{番号}` / `feat-issue-{番号}`
- コミットメッセージは日本語
- テストコードがある場合は必ず追加
