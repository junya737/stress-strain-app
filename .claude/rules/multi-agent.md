# Multi-Agent Coordination

Claude Code / Codex CLI / Gemini CLI の3エージェント協調ルール。

## エージェント構成

| エージェント | 役割 | 権限 | 設定ファイル |
|---|---|---|---|
| **Claude Code** | オーケストレーター・実装者 | 読み書き・実行・Git操作 | `.claude/CLAUDE.md` |
| **Codex CLI** | 設計・分析アドバイザー | **読み取り専用**（提案のみ） | `AGENTS.md` |
| **Gemini CLI** | リサーチ・情報収集 | **読み取り専用**（情報提供のみ） | `GEMINI.md` |

## 委譲判断基準

Claude Codeがタスク種別を判断し、適切なエージェントに**自動委譲**する。

| タスク種別 | 担当 |
|---|---|
| コード実装・修正・リファクタリング | Claude Code |
| Git操作・コミット・PR | Claude Code |
| テスト作成・実行 | Claude Code |
| 設計判断・アーキテクチャ検討 | **Codex CLI** |
| 複雑なバグの根本原因分析 | **Codex CLI** |
| コードレビュー・差分レビュー | **Codex CLI** |
| アイデア出し・手法提案・ブレスト | **Codex CLI** |
| アルゴリズム設計・数学的検討 | **Codex CLI** |
| Webリサーチ・最新技術調査 | **Gemini CLI** |
| 論文・ドキュメントの要約 | **Gemini CLI** |
| 大規模コンテキスト分析 | **Gemini CLI** |
| 画像/図表の解釈 | **Gemini CLI** |

## 呼び出しコマンド

```bash
# Codex CLI（プロンプトは位置引数）
codex exec "プロンプト内容"

# Gemini CLI（プロンプトは -p フラグ）
gemini -p "プロンプト内容"
```

## プロンプト構造テンプレート

### Claude Code → Codex CLI

```
【目的】{1文でタスクの目的}
【コンテキスト】{関連コード・ファイルパス・現状}
【制約】{コーディング規約・既存パターン等}
【質問】{具体的な質問や依頼}
【期待出力】結論 / 根拠 / 代替案 / リスク の形式で回答してください
```

### Claude Code → Gemini CLI

```
【検索目的】{何を調べたいか}
【クエリ】{検索キーワード・テーマ}
【条件】{期間・言語・ソース制限}
【期待出力】要約 / 重要ポイント / 引用元URL の形式で回答してください
```

## ドメイン知識の渡し方

AGENTS.md / GEMINI.md にドメイン知識は含めない。
Claude Codeが `.claude/rules/kaggle/competition-info/` 等から必要な情報を読み取り、
プロンプトの【コンテキスト】セクションに含めて渡す。

## 結果の統合ルール

- Codex / Gemini から受け取った結果は、Claude Codeが内容を検証してから採用する
- 提案をそのまま実装せず、プロジェクトの規約・既存パターンとの整合性を確認する
- 矛盾する提案があった場合は、ユーザーに判断を仰ぐ

## 禁止事項

| エージェント | 禁止事項 |
|---|---|
| **Codex CLI** | ファイル作成・編集、Git操作、外部API呼び出し |
| **Gemini CLI** | ファイル作成・編集、設計判断の最終結論、コード実装 |
| **Claude Code** | Codex/Geminiの提案を検証なしに採用すること |
