# Development Workflow

このドキュメントは、タスクを実行する際の標準的な作業フローと、安全管理のためのチェックリストを定義します。

## 🚫 Critical Rules
- **Baseでの作業禁止**: 現在チェックアウトされている親ブランチ（mainやworktreeのルート）に直接コミットしないこと。
- **Worktree越境禁止**: `git worktree` 環境では、現在のディレクトリ外（`../`）のファイルを絶対に修正しないこと。
- **一括Add禁止**: `git add .` や `git add -A` は禁止。修正したファイルのみを個別にaddすること。
- **No Git Stash**: `git stash` は使用しないこと。
- **コンテキスト確認**: 破壊的なコマンドの前に、必ず `pwd` と `git branch` を確認すること。
---

## 🔄 Standard Workflow

エージェントは修正や機能追加を行う際、**必ずこの手順をTodoとして認識し**、実行してください。

### 1. Worktree Strategy & Preparation
- **基本動作**: **「現在チェックアウトされているブランチ」を親（Base）として扱うこと**。
  - ❌ `main` に切り替えないこと（Worktree環境のため）。
  - ❌ エージェント自身で `git pull` を行わないこと（リスク回避）。

- **Tool Usage**:
  - **Serena MCP**: プロジェクト固有の情報検索や操作が必要な場合は、積極的に `serena MCP` を使用して効率化を図ること。

- **開始前確認**:
  1. `CLAUDE.md`および`rules`を読み込む。
  2. **実行確認**: 修正を加える前にコードを実行し、現在の挙動を確認する（`pwd` 確認）。

### 2. Implementation
1. **ブランチ作成**: 現在のHEADから作業用ブランチを作成・移動する。
   - 命名: `feat-<name>` または `fix-<name>`
2. **コーディング**:
   - **範囲**: 現在のWorktree内のファイルのみを修正対象とする。
   - **影響確認**: 該当修正によって他の処理に問題が生じないか慎重に確認する。

### 3. Verification
1. **動作確認**:
   - コミット前に必ずコードを実行し、エラーがないことを確認する。
   - **debugモードで実行**: スクリプトに `--debug` フラグがある場合は、debugモードで動作確認すること。
   - 動作確認中にエラーが発見された際は、コミットせずに修正を続けること。
2. **状態確認**:
   - `pwd && git branch && git status` を実行。
   - 正しいWorktree、正しいブランチにいるか確認。

### 4. Commit
1. **Staging**: `git add <file>`（修正したファイルのみ）。
2. **Commit**: 以下の「コミットメッセージ規約」に従ってコミットする。

## 📝 Commit Message Guidelines

コミットメッセージは必ず以下の形式で記述してください。

### 1. 概要行 (Subject)
- **形式**: `type(scope): 概要`
  - **type**: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`
  - **scope**: 変更箇所（例: `preprocess`, `model`, `api`）
- **内容**: 50文字以内の日本語。命令形で記述（例：「～を追加」「～を修正」）。

### 2. 空行
- 概要行の後に必ず空行を入れる。

### 3. 本文 (Body)
- **内容**: 日本語で記述。
  - なぜその変更が必要だったか
  - どう実装したか（詳細）
- **形式**: 72文字幅で改行する。
- **参照**: 関連するIssue等があれば `Refs: #123` のように明記。

## ✅ Safety Checklists

コマンド実行時は以下を必ず確認してください。

- **ファイル編集の前**: `pwd` を実行。プロジェクトルート内にいるか確認。
- **スクリプト実行の前**: `pwd` を実行。ルートからの実行か確認。
- **Git操作の前**: `pwd && git branch && git status` を実行。意図したWorktreeとブランチか確認。
